# DevPods presents Hidden Passageways Museum

This is the Godot engine part of the overall project. For the Unity project, see [https://github.com/Gamkedo/hidden-passageways-museum-unity](https://github.com/Gamkedo/hidden-passageways-museum-unity).

## About

A DevPods collaboration project that presents the history of DevPods past and present projects. Explore two virtual museums housing exhibits for game projects made by its members, a hub world to browse every game made by the DevPods community.

This is a DevPods community project. It requires membership to contribute and collaboration takes place on its Discord server. If you're interested in joining, check out [DevPods.gg](https://devpods.gg/)!

## Setup

This is a brief overview on how to get set up. For more in-depth instructions, refer to the DevPods onboarding videos! If you're not sure where that is, feel free to ask around on the DevPods Discord server - you can always try #spawn-point or #human-help channels.

1. This is the Godot engine part of the project. Please make sure to download and install [Godot 4.7](https://godotengine.org/download/) on the platform of your choice (Windows, MacOS, Linux etc.).

2. In the #godot_unity-hidden-passageways-museum channel, ask for access to this github repo. Feel free to @ the project lead and provide your github username. When they see your request you will receive a contributor invitation by email. Check your email to find a link to view the invite and that will enable you to accept the invite from your github account.

3. Use [Github Desktop](https://desktop.github.com/download/) or any other Git tool, such as the official CLI or any other third-party client or IDE plugin. If you're new to games development or new to DevPods, it's generally recommended to use the official desktop app: [https://desktop.github.com/download/](https://desktop.github.com/download/). Again, if in doubt, ask for help in the Discord server!

4. Once you've done all of the above, you're ready to clone the repo. Download the files to a directory/folder of your choosing and that will be where you access the project in the Godot engine editor.


## Things to keep in mind

The target platform of this project is for web export (HTML5) so be sure to take some time to read Godot's documentation about it: [https://docs.godotengine.org/en/stable/tutorials/export/exporting_for_web.html](https://docs.godotengine.org/en/stable/tutorials/export/exporting_for_web.html). If you're unsure about anything, feel free to ask in the relevant channels on the Discord server! If in doubt, drop a message in the #human-help channel.
